### What You Document

#### 1. Product Brief (`product-brief.md`)
- Product description and value proposition
- Target audience and user personas
- Business goals and success metrics
- Key features and functionality
- Constraints and assumptions

#### 2. Software Requirements Specification (`srs.md`)
- **Functional Requirements**: What the system must do
- **Non-Functional Requirements**: Performance, security, scalability
- **System Requirements**: Hardware, software, dependencies
- **User Requirements**: User roles, permissions, workflows
- **External Interfaces**: Third-party integrations

#### 3. Database Design (`database-design.md`)
- **Entity-Relationship Diagrams**: Visual representation of data model
- **Tables & Columns**: Detailed schema with data types
- **Relationships**: Foreign keys, cardinality (1:1, 1:N, N:M)
- **Normalization**: Ensure data integrity (1NF, 2NF, 3NF, BCNF)
- **Indexes**: Performance optimization
- **Constraints**: Primary keys, unique constraints, check constraints

#### 4. API Specification (`api-specification.md`)
- **Endpoints**: URL paths and HTTP methods
- **Request Formats**: Headers, body, query parameters
- **Response Formats**: Success/error responses, status codes
- **Authentication**: Auth methods, token handling
- **Rate Limiting**: Request limits and throttling
- **Versioning**: API version strategy

#### 5. System Architecture (`system-architecture.md`)
- **Architecture Diagrams**: System components and interactions
- **Technology Stack**: Languages, frameworks, databases
- **Design Patterns**: Architectural patterns used
- **Deployment Architecture**: Servers, containers, cloud services
- **Security Architecture**: Authentication, authorization, encryption
- **Scalability Strategy**: Horizontal/vertical scaling approach

#### 6. User Stories (`user-stories.md`)
- **User Stories**: As a [role], I want [feature] so that [benefit]
- **Acceptance Criteria**: Conditions for story completion
- **Priority**: Must-have, should-have, nice-to-have
- **Estimates**: Story points or time estimates

### For Frontend Projects

#### 7. Design System (`design-specs/design-system.md`)
- **Color Palette**: Primary, secondary, accent colors (hex codes)
- **Typography**: Font families, sizes, weights, line heights
- **Spacing**: Margin, padding, gap values
- **Components**: Buttons, cards, forms, navigation
- **Breakpoints**: Mobile, tablet, desktop sizes
- **Animations**: Transitions, hover effects, loading states

#### 8. Wireframes (`design-specs/wireframes/`)
- Low-fidelity sketches or high-fidelity mockups
- Page layouts and component placement
- User flows and navigation paths
- Mobile and desktop versions

#### 9. UI Specifications (`design-specs/ui-specifications/`)
- Detailed specs for each page/section
- Component behavior and interactions
- Responsive design rules
- Accessibility requirements

## ğŸ› Troubleshooting

### CORS Error
- Check `CORS_ORIGIN` in backend `.env`
- Ensure it includes frontend URL
- Restart backend

### Session Not Persisting
- Check Redis is running: `redis-cli ping`
- Verify cookies in browser DevTools
- Ensure `credentials: 'include'` in fetch calls

### Cannot Connect to Backend
- Verify backend is running: `curl http://localhost:3000/api/health`
- Check `NEXT_PUBLIC_BACKEND_URL` in frontend `.env.local`
- Check firewall settings

### Database Connection Failed
- Check PostgreSQL is running
- Verify credentials in backend `.env`
- Run migrations: `pnpm migration:up`
