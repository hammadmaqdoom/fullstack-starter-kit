## Design Instructions

[Specific notes]

Example:
```
- Keep answers concise (2-4 sentences ideal)
- Order questions by importance/frequency
- Most common questions should be first
- Avoid marketing fluff in answers - be direct
- If answer is long, consider breaking into multiple FAQs
- Link to relevant docs/pages within answers if helpful
- Ensure mobile tap targets are large enough (44px+)
- Don't auto-expand all FAQs - let users discover
- Consider adding "Was this helpful?" feedback (optional)
```
